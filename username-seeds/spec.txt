# Codex Handoff — Time-Derived Username Generator

## Objective

Build a small static webpage that derives a compound username from the visitor’s current local time.

The username is formed from:

* One word selected from a 10-item `A` list
* One word selected from a 14-item `B` list

This produces:

[
10 \times 14 = 140
]

possible usernames.

The webpage should run entirely client-side and be suitable for hosting with GitHub Pages.

---

## Core Derivation Method

Read the visitor’s current local hour and minute.

Let:

[
H = \text{hour from 0 to 23}
]

[
M = \text{minute from 0 to 59}
]

Convert the current time into the number of minutes since midnight:

[
T = 60H + M
]

Therefore:

[
0 \leq T \leq 1439
]

Map that value into the 140-name space:

[
N = T \bmod 140
]

Then derive the two list indices:

[
A_{\text{index}} = \left\lfloor \frac{N}{14} \right\rfloor
]

[
B_{\text{index}} = N \bmod 14
]

The resulting username is:

[
\text{username} = A[A_{\text{index}}] + B[B_{\text{index}}]
]

Do not insert spaces, underscores, or hyphens.

---

## Important Mathematical Requirement

Do not calculate both list indices directly from:

```js
HHMM % 10
HHMM % 14
```

Because 10 and 14 share a factor of 2, that approach only reaches 70 of the 140 possible pairs.

The implementation must use the flattened 140-item index method described above.

Because one day has 1,440 minutes:

[
1440 = 10(140) + 40
]

Each pairing appears either 10 or 11 times per day.

## Live Updating

The page should update automatically when the minute changes.

Do not unnecessarily recompute every animation frame.

Recommended behavior:

1. Generate immediately on page load.
2. Check the clock periodically, such as once per second.
3. Only update the derived username when the minute value changes.

The displayed seconds are optional. The derivation itself must only use hour and minute.

---

## Freeze Function

Add a button that toggles between:

```text
Freeze
Resume
```

When frozen:

* Keep the currently generated username visible.
* Stop replacing it when the time changes.
* Show the exact frozen timestamp.
* The copy button should continue to work.

When resumed:

* Immediately derive the username from the current local time.

This allows the user to keep a generated name on screen while creating an account.

---

Include a fallback for browsers where `navigator.clipboard` is unavailable.

## JavaScript Structure

Use small, clear functions.

Suggested structure:

```js
function validateWordLists() {}

function getMinutesSinceMidnight(date) {}

function deriveUsernameFromMinutes(minutesSinceMidnight) {}

function deriveUsernameFromDate(date) {}

function renderResult(result, date, mode) {}

function copyUsername() {}

function freezeResult() {}

function resumeLiveMode() {}

function startClock() {}
```

Suggested result object:

```js
{
  username,
  minutesSinceMidnight,
  flatIndex,
  aIndex,
  bIndex,
  wordA,
  wordB
}
```

Core function:

```js
function deriveUsernameFromMinutes(minutesSinceMidnight) {
  const totalCombinations = wordsA.length * wordsB.length;
  const flatIndex = minutesSinceMidnight % totalCombinations;
  const aIndex = Math.floor(flatIndex / wordsB.length);
  const bIndex = flatIndex % wordsB.length;

  return {
    username: `${wordsA[aIndex]}${wordsB[bIndex]}`,
    minutesSinceMidnight,
    flatIndex,
    aIndex,
    bIndex,
    wordA: wordsA[aIndex],
    wordB: wordsB[bIndex]
  };
}
```

## Sequence Behavior

The flattened order should be:

```text
A0+B0
A0+B1
...
A0+B13
A1+B0
A1+B1
...
A9+B13
```

After the final pair, it returns to:

```text
A0+B0
```

Do not randomize or shuffle the arrays at runtime.

The result must be deterministic:

* Same local hour and minute
* Same word-list order
* Same generated username

## Acceptance Criteria

The implementation is complete when:

* The page derives a username from the current local hour and minute.
* It calculates minutes since midnight as `hour * 60 + minute`.
* It maps the result through one flat 140-item cycle.
* All 140 A–B combinations are reachable.
* The generated result updates when the minute changes.
* The user can freeze and resume the result.
* The user can copy the username.
* The user can test an arbitrary time.
* Invalid word-list lengths produce a visible error.
* The app works locally without a server.
* The app deploys successfully on GitHub Pages.
* No external network requests occur.
* The README explains the derivation and deployment.
